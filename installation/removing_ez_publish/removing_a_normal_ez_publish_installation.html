Removing a normal eZ publish installation

These documentation pages are no longer maintained. Please visit the new documentation site.

This section describes how to completely remove an eZ publish installation (and all data that belongs to it). WARNING! Following the steps described here will destroy/remove ALL data (images, design, database, etc.), use with caution!

Removing eZ publish is done in four steps:

- eZ publish directory removal
- Database destruction
- Apache reconfiguration (optional)
- Cron job removal (optional)

Removing the eZ publish directory
Simply remove the eZ publish directory using your favourite tool. On Linux/UNIX systems, this would most likely be carried out using the rm (short for remove) command: 

$ rm -Rf /path/to/ez-publish-directory

Some file/directory permissions might be messed up; sometimes this prevents a regular user from removing all eZ publish files. In this case, in order to remove these files/directories, you'll need to gain root access (or ask someone with root privileges).

Windows users may simply delete the eZ publish directory using explorer.

Destroying the database

MySQL:

1. Start the MySQL client, log in using your username and password:

 

$ mysql -u <username> -p

 

If the username/password is correct, the client will then present a "mysql>" prompt.

 

2. Delete/remove the database using the drop command followed by the name of the database used for eZ publish:

 

mysql> drop database <database-name>;

PostgreSQL:

 

Remove the database by executing the PostgreSQL dropdb command from shell: 

 

$ dropdb <database-name>

Reconfiguring Apache (optional)
If virtual hosts were used (in order to gain access to different eZ publish sites/modules), it is likely that the Apache configuration file contains eZ publish specific settings. These are obviolusly not needed anymore, they can be safely removed. Using your favourite editor (notepad, wordpad, vi, emacs, ed, nano, pico), open the httpd.conf file, scroll down to the bottom and remove the eZ publish specific virtual host settings. Remember to restart Apache after altering the configuration file.

Removing cron jobs (optional)
Windows users should skip this part. If cron was configured to run eZ publish specific jobs, then these will have to be removed. You may have to edit a global cron file (residing under /etc/cron*) or use the crontab command, with the -e parameter (to edit a users private cron file). Remove the eZ publish specific entries.

Thats it.
