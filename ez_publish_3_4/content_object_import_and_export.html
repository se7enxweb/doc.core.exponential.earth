Content object import and export

These documentation pages are no longer maintained. Please visit the new documentation site.

From version 3.4 of eZ publish it is possible to import and export subtrees and nodes. To use this this feature, enter the admin interface, and go to Set up- Packages. It is also possible til use a command line version :

Content object export with ezpm

Press Create package to start package wizard.

Choose Content object export and press Create package.

Browse and select the subtrees and nodes you wish to export, and press Next.

Set the export options for the package, and press Next.

  • Include class definitions will include class definitions in the package. If the classes already exists on the target system, these packages will be ignored.
  • Include templates related exported objects will include all templates used by the exported objects in the selected site accesses. This will also include all fetch_aliases used

Please note that that related objects are ignored in the export in 3.4 release. This will be implemented later.

The next step is for specifying information about the package, press Next to continue.

Enter information about the package creator, and press Next.

The final step is for specifying changes done in the package compared to previous release. Press Finish to generate the package.

The package is now created and can be exported and installed on the system. ( Installing the package on the system makes no sense, since it already exists there. )
